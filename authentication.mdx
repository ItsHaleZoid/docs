Asimov API uses Bearer token authentication. You can authenticate using either:

1. **API Keys** (recommended for server-side applications)
2. **JWT Tokens** (for web applications using Supabase auth)

## Using API Keys

API keys are the recommended method for server-side applications. They start with the prefix `asmv-`.

### Generating an API Key

1. Navigate to [Settings > API](https://asimov.mov/settings/api)
2. Click **"Generate"** to create a new API key
3. Copy the key immediately (it's shown only once)

### Using API Keys in Requests

Include your API key in the `Authorization` header:

```
Authorization: Bearer asmv-YOUR_API_KEY_HERE
```

<CodeGroup>

```bash Example
curl -X POST https://asimov.mov/api/sources/content/add \
  -H "Authorization: Bearer asmv-YOUR_API_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{"content": "Your content here"}'
```

```javascript JavaScript
const response = await fetch('https://asimov.mov/api/sources/content/add', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer asmv-YOUR_API_KEY_HERE',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    content: 'Your content here'
  })
});
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer asmv-YOUR_API_KEY_HERE',
    'Content-Type': 'application/json'
}

response = requests.post(
    'https://asimov.mov/api/sources/content/add',
    headers=headers,
    json={'content': 'Your content here'}
)
```

</CodeGroup>

## Using JWT Tokens

For web applications using Supabase authentication, you can use JWT tokens directly:

```
Authorization: Bearer YOUR_JWT_TOKEN
```

The JWT token is automatically obtained when users authenticate through the web interface.

## Security Best Practices

<Warning>
  **Never commit API keys to version control.** Use environment variables or secure secret management systems.
</Warning>

- Store API keys in environment variables
- Rotate API keys regularly
- Use different API keys for different environments (development, staging, production)
- Revoke compromised keys immediately
- Never expose API keys in client-side code

## Error Responses

If authentication fails, you'll receive a `401 Unauthorized` response:

```json
{
  "error": "Invalid API key"
}
```

Common causes:
- Missing `Authorization` header
- Invalid or expired API key
- Incorrect header format (must be `Bearer <token>`)
