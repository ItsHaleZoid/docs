Parameters are custom metadata fields that allow you to filter and organize your content effectively. They enable powerful querying capabilities when searching your knowledge base.

<Warning>
  Parameters must be created in the dashboard first before they can be used in API requests. Parameter management is not available via API keys for security reasons.
</Warning>

## Using Parameters

### Adding Content with Parameters

When adding content via the API, you can include parameters that were previously created in the dashboard:

```bash
curl -X POST https://api.asimov.mov/add \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Documentation for version 2.0",
    "name": "Product Docs v2",
    "params": {
      "version": "2.0",
      "category": "documentation",
      "status": "published"
    }
  }'
```

<Warning>
  Parameters must be created first in the dashboard before they can be used. If you try to add content with parameters that don't exist, you'll receive an error.
</Warning>

### Searching with Parameters

Filter search results by parameters:

```bash
curl -X POST https://api.asimov.mov/search \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "documentation",
    "limit": 10,
    "params": {
      "version": "2.0",
      "category": "documentation"
    }
  }'
```

### Parameter Filtering Logic

- **All parameters must match**: All specified parameters must match exactly
- **Post-filtering**: Parameters are applied after vector search but before reranking
- **Case-sensitive values**: Parameter values are matched exactly as stored

## Creating Parameters

Parameters are created and managed through the Asimov dashboard at [https://asimov.mov/dashboard](https://asimov.mov/dashboard). Navigate to the Sources section to create, update, or delete parameters.

## Best Practices

1. **Plan Your Schema**: Design your parameter structure before adding content
2. **Use Consistent Values**: Use standardized values for better filtering
3. **Document Parameters**: Provide clear descriptions for each parameter in the dashboard
4. **Don't Over-filter**: Using too many parameters can reduce search results
5. **Version Your Content**: Use version parameters to track document versions

## Example Use Cases

### Version Control

```json
{
  "params": {
    "version": "2.0",
    "release_date": "2024-01-15"
  }
}
```

### Categorization

```json
{
  "params": {
    "category": "documentation",
    "topic": "authentication",
    "language": "english"
  }
}
```

### Status Tracking

```json
{
  "params": {
    "status": "published",
    "reviewed": "true",
    "author": "john-doe"
  }
}
```
